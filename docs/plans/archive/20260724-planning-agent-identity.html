Add a provider-neutral definition canary and a cost-bounded Codex planning campaign. A display nickname never counts as agent identity.

01 Trust boundary

The capability manifest proves which file was installed. The canary proves that the installed definition reached a completed child context. Neither a nickname nor parent narration is sufficient.

04 Verification

Formatter, lint, type checking, compilation, HomeOps build, Compose validation, and both provider campaign previews run without unit tests. Exact-footer parser smoke checks cover Codex and Claude, while the heavy scenario verifies API and banner wiring through their entry-point import graphs without executing agent-authored server code. The first live heavy run completed in 299.9 seconds with 1,526,455 total tokens but emitted no spawn or definition-canary evidence, correctly identifying a planning-instruction failure despite its long duration. The calibration also exposed and repaired a missing predictor field before final verification.
