Make event evidence an explicit case contract

Every new eval case must name the agent evidence it relies on. The harness rejects unsupported CLI/profile combinations before spending an agent call and records parser coverage so newly introduced CLI event shapes cannot disappear silently.

01 Success criteria

InvariantEvidence
Cases declare dependenciesrequired_evidence and its must-observe subset are mandatory at campaign, run, and hosted-dataset validation boundaries.
Unsupported cases fail before executionThe selected CLI profile advertises semantic parser capabilities and preflight reports unsupported case requirements.
Schema drift is visibleEach result stores observed, normalized, intentionally ignored, and unknown event-type sets.
Absence has explicit meaningMissing must-observe evidence fails the evidence contract; supported-only evidence can remain absent when the case is testing restraint.
Cost stays boundedStatic checks and one fixture-backed, single-arm agent invocation verify the final trace and metrics.

02 Contract flow

case.required_evidence + required_observed_evidence
        │
        ├── unsupported by selected CLI ── fail preflight, zero agent calls
        │
        └── supported
              │
              ├── must observe ── missing evidence fails the evidence contract
              └── support only ── absence can be meaningful agent behavior

CLI JSONL ── classify ── observed / normalized / ignored / unknown
                                                      │
                                                      └── MLflow trace + diagnostic metric
Unknown means the CLI schema changed or the parser lacks an explicit classification. It is diagnostic rather than an automatic behavioral failure, because a lifecycle event may be harmless while still requiring harness maintenance.

03 Implementation

  1. Define a shared semantic evidence vocabulary and per-profile support matrix.
  2. Add mandatory support and must-observe requirements to runtime, campaign, dataset, and case boundaries.
  3. Classify Codex and Claude JSONL event shapes and tool discriminators without retaining raw payloads.
  4. Attach coverage to AgentResult, predictor output, and agent.invoke.
  5. Emit evidence-contract status and unknown_agent_event_type_count as universal metrics.
  6. Document how future cases select requirements and how unsupported model-selection evidence is surfaced.
